## 🔥 Ultimate Coding Interview Strategy

### **1️⃣ Understanding the Problem Clearly**
- **Repeat the problem:** Paraphrase to ensure understanding
- **Clarify edge cases:** Input size, negative values, duplicates, empty inputs
- **Ask about input/output format:** Array, linked list, string, etc.
- **Confirm assumptions:** Valid input? Sorted? Memory constraints?
- **Walk through examples manually**

---

### **2️⃣ Breaking Down the Solution**
- **Brute force first:** Describe the basic O(n²) or naive approach
- **Optimize:** Sorting, hash maps, heaps, DP, binary search, two pointers, sliding window
- **Choose the best algorithm** based on constraints

---

### **3️⃣ Writing Clean & Efficient Code**
- Function signature with proper types
- Inline comments for clarity
- Handle edge cases explicitly
- Modular helper functions
- Meaningful variable names

---

### **4️⃣ Explaining the Code**
- Walk through step-by-step before running
- State time complexity: O(n), O(log n), O(n log n)
- State space complexity: O(1), O(n)
- Discuss trade-offs

---

### **5️⃣ Testing**
- Base cases (smallest input)
- Normal cases (expected input)
- Edge cases (empty, large, negative, duplicates)
- Off-by-one error checks

---

### **6️⃣ Debugging**
- Print debug statements at key steps
- Manually trace with sample input
- Use pen & paper for recursion/complex loops

---

## 🔍 Key Questions Before Coding

1. What is the expected input and output?
2. Can I assume the input will always be valid?
3. What edge cases should I consider?
4. What is the expected time/space complexity?
5. Should I prioritize speed or memory optimization?
6. Can I use extra data structures or optimize in-place?
7. Are there multiple valid solutions?
8. Do I need to handle parallel/concurrent processing?
9. Is recursion allowed, or should I use iteration?
10. How should I handle large datasets?
