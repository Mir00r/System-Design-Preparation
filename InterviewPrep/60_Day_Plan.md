# 📅 60-Day Interview Prep Plan: Balanced and Thorough

> *"60 days gives you time to build deep understanding rather than just surface familiarity. You can afford to practice more problems, do more mocks, and build genuine intuition for patterns."*

**⏱️ Estimated Time**: 10 minutes | **🎯 Difficulty**: 🟡 Medium

---

## 📋 Phase Overview

```
PHASE 1 (Days 1-20):  Foundation Building
  - Core data structures and algorithms
  - System design building blocks
  - 2-3 coding problems/day

PHASE 2 (Days 21-40): Pattern Mastery + Design Practice
  - Advanced patterns (DP, Graphs, Backtracking)
  - System design case studies (1 per 2 days)
  - 2-3 coding problems/day + mock interviews

PHASE 3 (Days 41-60): Mock Interviews + Polish
  - 3-4 mock interviews per week
  - Weak area deep-dives
  - Behavioral preparation
  - Company-specific preparation

DAILY SCHEDULE (2-3 hours):
  Coding:     1-1.5 hrs (2-3 problems)
  Design:     45 min (study or practice)
  Behavioral: 15 min (story prep or review)
```

---

## 📅 Week-by-Week Breakdown

```
WEEKS 1-2: Data Structure Fundamentals
  Arrays, Strings, HashMap, Two Pointers, Sliding Window
  Stack, Queue, Linked Lists
  System Design: Caching, Load Balancing, CDN, Databases
  Target: 30-40 problems solved

WEEKS 3-4: Trees, Graphs, Binary Search
  BST, BFS/DFS, Shortest Path, Topological Sort
  Binary Search variations, Heap/Priority Queue
  System Design: URL Shortener, Twitter, Rate Limiter
  Target: 30-40 more problems (60-80 total)

WEEKS 5-6: Dynamic Programming, Backtracking, Greedy
  1D/2D DP patterns, Knapsack, Intervals
  Permutations, Combinations, Constraint satisfaction
  System Design: WhatsApp, Netflix, Uber, Payment System
  Target: 30-40 more problems (90-120 total)

WEEKS 7-8: Mocks + Polish
  3-4 full mock interviews per week
  Review all weak areas identified in mocks
  Behavioral story refinement (10 STAR stories)
  Company research + question preparation
  Target: 8-12 mock interviews completed

TOTAL PROBLEMS: 100-150 (mix of Easy/Medium/Hard: 20/60/20%)
```

---

## 📊 Weekly Milestones

| Week | Coding | System Design | Behavioral |
|---|---|---|---|
| 1-2 | Easy in 10 min, Medium in 25 min | Know all building blocks | 3 STAR stories |
| 3-4 | Medium in 20 min | Design simple systems | 6 STAR stories |
| 5-6 | Approach Hard problems | Design complex systems | 10 STAR stories |
| 7-8 | Consistent performance in mocks | Confident in any SD problem | Natural delivery |

---

## 🎯 Key Differences from 30-Day Plan

```
MORE DEPTH:
  - 2x more coding problems (better pattern recognition)
  - More system design practice (6-8 full designs vs 3-4)
  - Time for spaced repetition (re-solve problems after 1 week)
  
MORE PRACTICE:
  - Start mocks earlier (week 5 instead of week 4)
  - More mock interviews total (8-12 vs 3-4)
  - Time to iterate on feedback from mocks
  
LESS RUSHED:
  - Can afford to spend extra day on difficult topics
  - Time to read DDIA or Alex Xu's book cover-to-cover
  - Rest days built in (1 per week) for retention
```
