# 🎭 Mock Interviews: Practice That Simulates the Real Thing

> *"You wouldn't give a presentation without rehearsing. Mock interviews are rehearsal for the real thing — they build muscle memory for thinking out loud, managing time, and handling pressure."*

**⏱️ Estimated Time**: 15 minutes | **🎯 Difficulty**: 🟢 Beginner | **🔗 Prerequisites**: [How to Crack System Design](./How_To_Crack_System_Design.md)

---

## 🎯 Why Mock Interviews Matter

```
SKILLS THAT ONLY IMPROVE WITH PRACTICE:
  ✅ Thinking out loud (feels unnatural at first)
  ✅ Time management (45 min goes fast under pressure)
  ✅ Handling being stuck (without panic)
  ✅ Drawing diagrams while explaining
  ✅ Asking good clarifying questions
  ✅ Whiteboard/shared editor fluency
  
  Reading about interviews ≠ being good at interviews
  Knowledge × Practice = Performance
```

---

## 📋 Mock Interview Structure

```
SYSTEM DESIGN MOCK (45 min):
  Interviewer picks a problem (e.g., "Design Instagram")
  Candidate follows the 4-step framework
  Interviewer gives hints/redirects as real interviewer would
  
  After: 15 min debrief
    - What went well?
    - What was missed?
    - Did you ask good questions?
    - Did you draw clear diagrams?
    - Time management?

CODING MOCK (45 min):
  Interviewer picks 1-2 problems (medium difficulty)
  Candidate solves on shared editor (no IDE autocomplete!)
  Interviewer observes: communication, approach, code quality
  
  After: 15 min debrief
    - Did you clarify requirements?
    - Did you plan before coding?
    - Did you test your solution?
    - Code readability?

BEHAVIORAL MOCK (30 min):
  Interviewer asks 3-4 behavioral questions
  Candidate uses STAR method
  Interviewer probes for depth
  
  After: 10 min debrief
    - Were answers specific or generic?
    - Did you quantify impact?
    - Were stories relevant to the question?
```

## 🛠️ How to Run Mock Interviews

```
OPTION 1: With a friend/colleague
  - Trade: you interview them, they interview you
  - Use problems from this repo's SystemDesignCaseStudies/
  - Be honest in feedback (kindness ≠ avoiding truth)

OPTION 2: Online platforms
  - Pramp (free, peer-to-peer)
  - Interviewing.io (anonymous, with real interviewers)
  - Exponent (community + coaches)
  
OPTION 3: Self-mock (solo practice)
  - Set timer for 45 minutes
  - Pick a problem you haven't solved
  - Talk out loud (record yourself!)
  - Review recording: "Did I communicate clearly?"

FREQUENCY:
  Minimum: 1 mock/week during active prep
  Ideal: 2-3 mocks/week in final 2 weeks before interviews
  Mix: alternate between system design, coding, and behavioral
```

## 📊 Self-Assessment Rubric

| Criteria | 1 (Needs Work) | 3 (Good) | 5 (Excellent) |
|---|---|---|---|
| Communication | Silent, mumbles | Explains approach | Clear, structured narrative |
| Requirements | Dives straight in | Asks 1-2 questions | Thorough scoping |
| Trade-offs | Single solution | Mentions alternatives | Deep analysis with reasoning |
| Time management | Runs out of time | Finishes rushed | Paced well, covers depth |
| Edge cases | Ignores them | Mentions a few | Systematically addresses |
