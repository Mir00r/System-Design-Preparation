# 📅 30-Day Interview Prep Plan: Intensive Sprint

> *"30 days is enough to prepare for interviews if you're already a working engineer. This plan assumes 3-4 hours/day of focused study. Prioritize ruthlessly — you can't cover everything, so cover the highest-impact topics."*

**⏱️ Estimated Time**: 10 minutes | **🎯 Difficulty**: 🟡 Medium

---

## 📋 Weekly Overview

```
WEEK 1: Foundations (coding patterns + core concepts)
WEEK 2: System Design (building blocks + practice)
WEEK 3: Advanced (hard problems + deep dives)
WEEK 4: Mock Interviews + Polish (simulation + review)

DAILY SCHEDULE (3-4 hours):
  Morning (1.5 hr):  Coding practice (2-3 problems)
  Afternoon (1 hr):  System Design study
  Evening (0.5 hr):  Behavioral prep + review
```

---

## 📅 Day-by-Day Plan

```
═══════════════════════════════════════════════════════════════════
WEEK 1: CODING FOUNDATIONS
═══════════════════════════════════════════════════════════════════

Day 1:  Arrays + HashMap — Two Sum, Contains Duplicate, Group Anagrams
        System Design: Read BuildingBlocks/Caching.md
        
Day 2:  Two Pointers — Valid Palindrome, 3Sum, Container With Water
        System Design: Read BuildingBlocks/LoadBalancing.md
        
Day 3:  Sliding Window — Max Subarray, Longest Substring, Min Window
        System Design: Read KeyConcepts/CAPTheorem.md
        
Day 4:  Stack/Queue — Valid Parentheses, Min Stack, Daily Temperatures
        System Design: Read Database/SQL_Vs_NoSQL.md
        
Day 5:  Linked Lists — Reverse, Merge Two, Detect Cycle, Remove Nth
        System Design: Read BuildingBlocks/MessageQueues.md
        
Day 6:  Binary Search — Search Rotated Array, Find Min, Search Matrix
        System Design: Read BuildingBlocks/APIGateway.md
        
Day 7:  REVIEW + BEHAVIORAL — Review weak problems, prepare 3 STAR stories
        System Design: Read BuildingBlocks/RateLimiting.md

═══════════════════════════════════════════════════════════════════
WEEK 2: TREES, GRAPHS + SYSTEM DESIGN
═══════════════════════════════════════════════════════════════════

Day 8:  Trees (BFS/DFS) — Max Depth, Invert, Level Order, Validate BST
        System Design: Design URL Shortener (practice)
        
Day 9:  Trees (advanced) — LCA, Serialize/Deserialize, Path Sum
        System Design: Read Database/Sharding.md
        
Day 10: Graphs (BFS) — Number of Islands, Rotten Oranges, Word Ladder
        System Design: Read KeyConcepts/Consistent_Hashing.md
        
Day 11: Graphs (DFS) — Clone Graph, Course Schedule, Pacific Atlantic
        System Design: Design Twitter Feed (practice)
        
Day 12: Heap/Priority Queue — Top K Frequent, Merge K Lists, Median
        System Design: Read Microservices/ patterns
        
Day 13: Trie + Union-Find — Implement Trie, Word Search, Accounts Merge
        System Design: Design WhatsApp (practice)
        
Day 14: REVIEW + MOCK INTERVIEW — Full coding mock (45 min, timed)
        Behavioral: Prepare 3 more STAR stories

═══════════════════════════════════════════════════════════════════
WEEK 3: DP, BACKTRACKING + ADVANCED DESIGN
═══════════════════════════════════════════════════════════════════

Day 15: DP (1D) — Climbing Stairs, House Robber, Coin Change
        System Design: Read Cloud/AWS/Core_Services.md
        
Day 16: DP (2D) — Unique Paths, Longest Common Subsequence
        System Design: Design Netflix (practice)
        
Day 17: Backtracking — Permutations, Combinations, Subsets, N-Queens
        System Design: Read Security/OAuth2.md
        
Day 18: Greedy — Jump Game, Task Scheduler, Meeting Rooms
        System Design: Design Payment System (practice)
        
Day 19: Intervals + Sorting — Merge Intervals, Insert Interval
        System Design: Read Observability/ concepts
        
Day 20: Bit Manipulation + Math — Single Number, Power of Two
        System Design: Design Uber (practice)
        
Day 21: FULL MOCK DAY — System design mock (45 min) + Coding mock (45 min)
        Review feedback, identify gaps

═══════════════════════════════════════════════════════════════════
WEEK 4: MOCKS + POLISH
═══════════════════════════════════════════════════════════════════

Day 22: Review weak coding patterns + 2 hard problems
        Behavioral: Refine all STAR stories (10 stories ready)
        
Day 23: System Design mock (record yourself)
        Review company-specific info (culture, values, products)
        
Day 24: Coding mock (2 medium problems, 45 min)
        System Design: Review trade-offs and numbers
        
Day 25: Full behavioral mock (6 questions, STAR)
        Light coding review (re-solve favorites)
        
Day 26: System Design deep-dive on YOUR weak area
        Coding: 2 new problems (medium/hard)
        
Day 27: FINAL MOCK DAY — full interview simulation
        Morning: coding, Afternoon: system design, Evening: behavioral
        
Day 28: Review all mock feedback, patch gaps
        Prepare questions to ASK the interviewer
        
Day 29: Light review only — re-read notes, review patterns
        Rest and confidence building
        
Day 30: REST DAY before interview — light exercise, good sleep
        Trust your preparation!
```
