## 🔄 Interview Loops

```
TYPICAL FAANG ON-SITE (4-6 rounds, 1 day):

  Round 1: Coding (45 min) — Data structures & algorithms
  Round 2: Coding (45 min) — Different pattern from round 1
  Round 3: System Design (45-60 min) — Design a system
  Round 4: Behavioral (45 min) — Leadership, teamwork, conflict
  Round 5: Coding or Domain-Specific (varies)
  
  Lunch: NOT an interview (but be personable!)

TIMELINE:
  Recruiter Screen → Phone Screen (1-2 coding) → On-site (4-6 rounds)
  Total: 4-8 weeks from application to offer
  
  Phone screen: 1 LeetCode medium in 45 min
  On-site coding: 2 LeetCode mediums or 1 hard in 45 min
  System design: Design X (open-ended, no single right answer)
```

## 🏢 Company-Specific

```
GOOGLE:
  Focus: Algorithms, scalability, Googleyness
  Coding: Harder problems, emphasis on optimal time/space
  System Design: Google-scale (billions of users)
  Behavioral: "Googleyness" — collaborative, humble, driven
  Unique: Hiring committee decides (not just interviewer)
  
  Prep: LeetCode Hard, distributed systems, back-of-envelope math

META (FACEBOOK):
  Focus: Move fast, product thinking, coding speed
  Coding: Medium problems, emphasis on SPEED (2 problems in 45 min)
  System Design: Social graph, real-time, news feed
  Behavioral: "Move fast" culture fit, impact-driven
  Unique: Strong product sense expected (not just tech)
  
  Prep: Speed-code LeetCode mediums, design social features

AMAZON:
  Focus: Leadership Principles (14 LPs drive EVERYTHING)
  Coding: Standard DS&A, practical problems
  System Design: Scalable services, microservices
  Behavioral: STAR format answers mapped to Leadership Principles
  Unique: Bar Raiser interview (independent evaluator)
  
  Prep: Memorize 14 Leadership Principles, prepare 2 stories per LP

APPLE:
  Focus: Craftsmanship, attention to detail, domain expertise
  Coding: Clean, production-quality code
  System Design: Practical, detail-oriented
  Behavioral: Passion for products, collaboration
  Unique: More emphasis on domain knowledge (iOS, ML, etc.)
  
  Prep: Write polished code, know Apple's tech stack

NETFLIX:
  Focus: Senior-only hiring, cultural fit, autonomy
  Coding: Less emphasis (more system design for senior)
  System Design: Streaming, content delivery, recommendation
  Behavioral: Netflix culture (freedom & responsibility)
  Unique: No entry-level hiring, stock-only compensation model
  
  Prep: Read Netflix culture deck, distributed systems expertise
```

## 📊 Leveling

```
LEVEL EXPECTATIONS:

  L3/E3 (Junior — 0-2 years):
    Coding: Solve medium problems independently
    Design: Not expected (or very basic)
    Behavioral: Learning orientation, collaboration
    
  L4/E4 (Mid — 2-5 years):
    Coding: Solve medium quickly, approach hard
    Design: Design a single service/feature
    Behavioral: Independent delivery, mentoring others
    
  L5/E5 (Senior — 5-8 years):
    Coding: Solve hard problems, optimal solutions
    Design: Design full systems, discuss trade-offs deeply
    Behavioral: Technical leadership, cross-team impact
    
  L6/E6 (Staff — 8+ years):
    Coding: Less weight (but still tested!)
    Design: Design at company scale, influence architecture
    Behavioral: Org-wide impact, strategic thinking

COMPENSATION (US, total comp):
  L3: $150-200K
  L4: $200-300K
  L5: $300-450K
  L6: $450-700K
  L7: $700K-1.2M+
```

## 🎯 Preparation Strategy

```
8-WEEK FAANG PREP PLAN:

WEEKS 1-2: Foundation
  - Review data structures (arrays, trees, graphs, heaps)
  - Practice 3 easy + 2 medium LeetCode problems daily
  - Read "System Design Interview" Vol 1 (Alex Xu)

WEEKS 3-4: Pattern Mastery
  - Focus on the 15 core patterns (see Coding Rounds guide)
  - Practice 2-3 medium problems daily (timed: 25 min each)
  - Study 2 system design problems per week

WEEKS 5-6: Advanced + Mock Interviews
  - Practice hard problems (1 per day)
  - Full system design mocks (2 per week)
  - Prepare behavioral stories (STAR format, 10 stories)

WEEKS 7-8: Final Polish
  - Mock interviews (3-4 per week, all types)
  - Review weak areas from mocks
  - Company-specific prep (read engineering blogs, culture)
  - Rest and confidence building (don't cram last day!)
```

## ❓ Common Questions by Company

```
SYSTEM DESIGN (commonly asked):
  Google:  Design YouTube, Google Maps, Google Search, Gmail
  Meta:    Design Instagram Feed, Facebook Messenger, Live Video
  Amazon:  Design Amazon Product Page, Order System, S3
  Netflix: Design Netflix Streaming, Recommendation Engine

BEHAVIORAL THEMES:
  Amazon:  "Tell me about a time you... (mapped to LP)"
    - Customer Obsession: put customer first
    - Ownership: went beyond your role
    - Dive Deep: uncovered root cause
    - Bias for Action: decided with incomplete info
    
  Google:  
    - Navigated ambiguity
    - Influenced without authority
    - Failed and learned
    
  Meta:
    - Moved fast and broke things (then fixed them)
    - Built consensus across teams
    - Delivered measurable impact
```

## ⚠️ Common Mistakes

1. **Not preparing behavioral** — Engineers focus 90% on coding and ignore behavioral. At senior level, behavioral is 30-40% of the decision. Prepare 10+ STAR stories covering different themes.

2. **Practicing only easy problems** — FAANG asks mediums and hards. If you're only comfortable with easy, you'll struggle. Target: solve most mediums in 20 min, attempt hards within 30 min.

3. **Not researching the company** — Each company has unique culture/values. Amazon's Leadership Principles are CRITICAL. Google values collaboration. Netflix values autonomy. Tailor your answers.
