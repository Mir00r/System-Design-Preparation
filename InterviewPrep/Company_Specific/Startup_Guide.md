# 🚀 Startup Interview Guide: What Startups Look For

> *"Startup interviews differ from Big Tech. They value breadth over depth, shipping speed over perfect code, and practical problem-solving over algorithmic puzzles. They want builders who can wear multiple hats."*

**⏱️ Estimated Time**: 15 minutes | **🎯 Difficulty**: 🟢 Beginner | **🔗 Prerequisites**: [InterviewPrep README](../README.md)

---

## 🏢 How Startup Interviews Differ

```
BIG TECH vs STARTUP:

  ┌──────────────────────────┬─────────────────────────────────────────┐
  │ Big Tech                 │ Startup                                 │
  ├──────────────────────────┼─────────────────────────────────────────┤
  │ 4-6 formal rounds       │ 2-4 rounds (faster process)             │
  │ LeetCode hard           │ Practical coding (take-home or live)     │
  │ System design (theory)  │ System design (practical, their product) │
  │ Standard behavioral     │ Culture fit, hustle, adaptability        │
  │ Specialized role        │ Full-stack, wear multiple hats           │
  │ Weeks to decide         │ Days to decide (competing for talent)    │
  │ Structured rubric       │ "Would I want to work with this person?" │
  │ TC: $300-500K           │ TC: $150-250K + equity (lottery ticket)  │
  └──────────────────────────┴─────────────────────────────────────────┘
```

---

## 🎯 What Startups Evaluate

```
1. CAN YOU BUILD? (70% of the decision)
   - Take-home project or live pair programming
   - "Build a feature similar to what we're working on"
   - Clean code, working software, good architecture
   - Speed: can you ship in days, not weeks?

2. ARE YOU AUTONOMOUS? (20%)
   - Can you figure things out without hand-holding?
   - Do you ask good questions or wait to be told?
   - Will you take ownership end-to-end?

3. DO YOU FIT THE TEAM? (10%)
   - Small team = every person matters
   - Communication style, energy, collaboration
   - "Would I want to pair program with this person every day?"
```

## 📋 Typical Startup Interview Process

```
Stage 1: Recruiter/Founder call (30 min)
  - "Tell me about yourself" (elevator pitch)
  - "Why this startup?" (show you researched them!)
  - "What are you looking for?" (alignment check)

Stage 2: Technical screen (60 min) — ONE of:
  Option A: Take-home project (4-8 hours)
    "Build a simplified version of our feature X"
    "Submit code + README explaining your decisions"
  Option B: Live coding (pair programming style)
    "Let's build this feature together"
    More collaborative, less adversarial than Big Tech

Stage 3: System design / architecture (45-60 min)
  Often about THEIR product:
  "How would you redesign our notification system?"
  "We're scaling from 1K to 100K users — what changes?"
  Practical, grounded in real problems

Stage 4: Culture fit / founder chat (30-45 min)
  - Why startup over big company?
  - Tell me about something you built end-to-end
  - How do you handle ambiguity/changing priorities?
  - What would you do in your first 30 days?
```

## 💡 Tips for Startup Interviews

```
1. RESEARCH THE PRODUCT
   Use their product before the interview
   "I noticed your onboarding flow does X — I'd suggest Y because..."
   Shows initiative and product thinking

2. SHOW BREADTH
   Startups need generalists:
   "I've done frontend React, backend Java, deployed to AWS, 
    set up CI/CD, and handled on-call incidents"

3. EMPHASIZE SHIPPING
   "In my last role, I took a feature from idea to production in 2 weeks"
   Speed matters more than perfection at early-stage startups

4. ASK SMART QUESTIONS
   "What's your biggest technical challenge right now?"
   "How do you balance tech debt vs feature velocity?"
   "What does success look like for this role in 90 days?"

5. SHOW OWNERSHIP MINDSET
   "I'd own the entire feature: design, implement, test, deploy, monitor"
   Not: "I'd wait for the architect to tell me what to build"
```

## ⚠️ Red Flags to Watch For (as a candidate)

```
  🚩 No clear engineering culture (no tests, no code review, no CI/CD)
  🚩 Founders can't articulate technical vision
  🚩 "We're looking for a 10x developer" (translation: work 80 hrs/week)
  🚩 Equity with no vesting details or cap table transparency
  🚩 Unrealistic timelines ("rebuild everything in 3 months")
  🚩 High engineer turnover (check LinkedIn for tenure)
```
